GIT commands and procedures for using GIT & GitHub

Procedure to clone a GitHub repo and push file(s)/changes to that GitHub repo.

  1. git clone 'GitHub SSH key from GitHub repo page'. This creates a directory that mirrors the GitHub repo
  2. Create your content i.e. HTML, CSS, JS, etc
  3. git add 'filename' or '.' for all files. This adds the file(s) for staging.
  4. git commit -m 'reason for push to GitHub'
  5. git checkout -b feature/'HTML or CSS or JS or whatever'. This creates a new branch (-b) and switches to it.
  6. git push --set-upstream origin feature/html(or whatever). Pushes the changes to the specified branch on GitHub, if the branch already exists on GitHub remove the --set-upstream option

Once a repo and local directory has been established, synched, and pushed, the following is all that is required to nmake updates/changes:

  1. git add filename(s) or '.'
  2. git commit -m 'message regarding reasons for update(s)
  3. git push
